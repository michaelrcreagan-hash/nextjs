---
name: cbt:clear
description: Generate context handoff document before clearing conversation
argument-hint: ""
allowed-tools:
  - Read
  - Write
  - Bash
  - Glob
---

<objective>
Before the user clears conversation context, auto-generate a handoff document (.cbt/handoff.md)
that captures the current session state so the next Claude session can seamlessly continue.
Then instruct the user to run /clear.
</objective>

<execution_context>
@strategies/{active}/.cbt/state.yaml
@strategies/{active}/DISCOVERY.md (if exists)
@strategies/{active}/RESEARCH.md (if exists)
@strategies/{active}/EDA.md (if exists)
@strategies/{active}/BUILD_PLAN.md (if exists)
@strategies/{active}/config.yaml
</execution_context>

<process>

## 1. Find Active Strategy

Locate the strategy folder by finding .cbt/state.yaml.
Read current state to understand where things stand.

## 2. Gather Session Context

Collect information from all available sources:

### From state.yaml:
- Current phase
- Phases completed
- Mode (yolo/interactive) and engine (pandas/fast)
- Build progress (if in build phase)
- Current experiment (if in iterate phase)
- Best experiment results
- Pending observations

### From recent files:
- Last modified experiment in experiments/
- Recent observations in observations/
- Any partially built files in src/
- Report status

### From current conversation context:
- What was being worked on this session
- Any findings or results generated
- Open questions or hypotheses being tested
- Problems encountered
- Decisions made

## 3. Generate .cbt/handoff.md

Create/overwrite the handoff document:

```markdown
# Session Handoff

**Strategy:** {name}
**Date:** {current_date}
**Phase:** {current_phase}
**Mode:** {yolo/interactive} | **Engine:** {pandas/fast}

---

## Where We Left Off

{Description of what was being worked on when /cbt:clear was called}

## Current State

- **Phase:** {phase} ({what's done, what's next})
- **Build Progress:** {if applicable, e.g., "3/5 steps complete"}
- **Experiments:** {count} runs, best: {best_id} (Sharpe: {best_sharpe})
- **Pending Observations:** {count} ideas to test

## This Session's Work

### What Was Done
{Bullet list of what was accomplished this session}

### Key Findings
{Any important results, patterns, or insights discovered}

### Decisions Made
{Any configuration changes, approach shifts, or design decisions}

## Open Questions

{Questions or hypotheses that were being explored but not yet resolved}

## Problems / Blockers

{Any issues encountered that need attention}

## Suggested Next Steps

1. {Most immediate next action}
2. {Follow-up action}
3. {Optional further action}

## Suggested Command

```
/cbt:{next_command}
```

---

*Generated by /cbt:clear on {date}*
```

## 4. Output Summary

Display a summary to the user:

```
Handoff saved to .cbt/handoff.md

Session Summary:
  Phase: {phase}
  Work done: {brief description}
  Next: /cbt:{suggested_command}

Now run /clear to reset the conversation.

When you start a new session, run /cbt:status to pick up where you left off.
The handoff context will be automatically displayed.
```

</process>

<constraints>
- ALWAYS overwrite previous handoff.md (only latest matters)
- DO NOT delete or modify any other files
- DO NOT actually clear the conversation - just generate the handoff and tell the user to run /clear
- Be comprehensive but concise in the handoff - it should be quick to read
- Focus on actionable information, not verbose descriptions
</constraints>

<success_criteria>
- [ ] .cbt/handoff.md created with all relevant context
- [ ] Current session work captured accurately
- [ ] Open questions and problems documented
- [ ] Clear next steps suggested
- [ ] User instructed to run /clear
- [ ] Previous handoff overwritten (not accumulated)
</success_criteria>
