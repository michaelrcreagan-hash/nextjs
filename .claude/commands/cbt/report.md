---
name: cbt:report
description: Create or update a living project report
argument-hint: "[init|update|add <section>]"
allowed-tools:
  - Read
  - Write
  - Edit
  - Bash
  - Glob
---

<objective>
Create and maintain a comprehensive living REPORT.md that aggregates information from all project documents.
Auto-populates sections from existing files and can be updated as the project evolves.
</objective>

<execution_context>
@strategies/{active}/.cbt/state.yaml
@strategies/{active}/DISCOVERY.md (if exists)
@strategies/{active}/RESEARCH.md (if exists)
@strategies/{active}/EDA.md (if exists)
@strategies/{active}/BUILD_PLAN.md (if exists)
@strategies/{active}/DEEP_ANALYSIS.md (if exists)
@strategies/{active}/config.yaml
@strategies/{active}/experiments/
@strategies/{active}/observations/
@strategies/{active}/plots/
</execution_context>

<process>

## 1. Parse Arguments

- `init` → Create initial report from scratch
- `update` → Refresh all sections with latest data
- `add <section>` → Add a custom section (user provides content)
- No args → If report exists, update. If not, init.

## 2. Gather Data from All Sources

### From DISCOVERY.md:
- Strategy name and hypothesis
- Entry/exit logic summary
- Success/kill criteria

### From RESEARCH.md:
- Key findings and confidence rating
- Risk assessment
- References

### From EDA.md:
- Key statistical findings
- Data characteristics
- Plot references

### From config.yaml:
- Current configuration (account, sizing, risk, fees)
- Engine type

### From BUILD_PLAN.md:
- Build complexity and steps

### From experiments/:
- All experiment results (sorted by Sharpe or chosen metric)
- Best experiment highlighted
- Baseline comparison

### From observations/:
- All saved observations and hypotheses
- Which have been tested

### From DEEP_ANALYSIS.md:
- Forensic analysis highlights
- Recommendations status

### From plots/:
- List all generated plots with links

</process>

<constraints>
- DO auto-populate from all available documents
- DO preserve custom sections when updating
- DO handle missing documents gracefully (skip section, note as unavailable)
- DO NOT duplicate entire documents - summarize and link
- DO NOT overwrite custom sections on update
- DO keep the report readable and scannable
</constraints>

<success_criteria>
- [ ] REPORT.md created/updated with all available data
- [ ] All project documents referenced and summarized
- [ ] Experiment history complete and sorted
- [ ] Plots linked with relative paths
- [ ] Custom sections preserved on update
- [ ] State updated with report_file path
</success_criteria>
