<objective>
Guide deployment of a backtested strategy as a live trading bot.
Supports multiple exchanges, notification channels, and safety features.
Paper trading mode by default before live deployment.
</objective>

<execution_context>
@strategies/{active}/.cbt/state.yaml
@strategies/{active}/config.yaml
@strategies/{active}/strategy.py
@strategies/{active}/src/
@strategies/{active}/experiments/
@~/.claude/cbt-framework/templates/live/
</execution_context>

<principles>
- Safety first: paper trade before live
- Never hardcode credentials: always use .env
- Kill switch is mandatory
- Max position size limits required
- Cross-exchange support (data from one, signals to another)
- Notification on every trade and error
</principles>

<process>

## 1. Parse Arguments

- `setup` → Initial setup (exchange, API keys, notifications)
- `paper` → Start paper trading
- `live` → Switch to live trading (requires paper trade first)
- `status` → Show bot status
- `stop` → Stop running bot
- No args → Interactive setup flow

## 2. Pre-flight Checks

Before ANY deployment:
- [ ] Strategy has been backtested (experiments/ has results)
- [ ] Backtest results are acceptable (warn if Sharpe < 1.0)
- [ ] Config is finalized
- [ ] Data source is specified
- Warn user about risks of live trading

## 3. Setup Flow

### Step 1: Select Exchange
Ask user:
- **Bybit** - Derivatives + spot, good API, popular for crypto
- **Kraken** - Established, good security, spot + futures
- **Binance** - Largest volume, most pairs
- **Hyperliquid** - Decentralized perpetuals, on-chain

### Step 2: Configure API Credentials

Create `.env.template`:
```
# Exchange API Credentials
# NEVER commit this file to git!
EXCHANGE_API_KEY=your_api_key_here
EXCHANGE_API_SECRET=your_api_secret_here
EXCHANGE_SUBACCOUNT=  # Optional

# Notification Credentials (optional)
DISCORD_WEBHOOK_URL=
TELEGRAM_BOT_TOKEN=
TELEGRAM_CHAT_ID=
TWILIO_SID=
TWILIO_TOKEN=
TWILIO_FROM=
TWILIO_TO=
EMAIL_USER=
EMAIL_PASS=
EMAIL_TO=
```

Add to `.gitignore`:
```
.env
.env.local
```

### Step 3: Set Up Notifications
Ask which channels to enable:
- **Discord** (webhook) - Easiest setup
- **Telegram** (bot API) - Most popular for trading
- **SMS** (Twilio) - For critical alerts
- **Email** (SMTP) - Traditional
- **None** - Skip notifications

### Step 4: Safety Parameters
Configure:
- Max position size (absolute limit)
- Kill switch drawdown threshold (default: 10%)
- Max daily trades (rate limit)
- Error retry count

## 4. Generate Bot Code

Based on exchange choice, generate from templates:

### Core Files:
- `live_bot.py` - Main bot entry point
- `exchange_client.py` - Exchange-specific API client (from template)

## 5. Paper Trading Mode

When `paper` mode:
- Connect to exchange for real data
- Simulate order execution (no actual orders)
- Track virtual portfolio
- Log all "trades" to `trades/paper_*.csv`
- Full notifications as if live
- Run for recommended minimum period before going live

## 6. Live Trading Mode

Requirements before live:
- [ ] Paper traded for minimum period
- [ ] Paper results acceptable
- [ ] User explicitly confirms understanding of risks
- [ ] Kill switch configured
- [ ] Notifications working (send test)

## 7. Update State

```yaml
  exchange: {exchange}
  mode: {paper|live}
```

Update config.yaml live section.
